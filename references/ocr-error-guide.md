# OCR 错别字判别指南

这份文档是 SKILL.md 的延伸，给出更细的判别示例和边界情况。核心精神不变：
**少改、拿不准就不改**。读完只需记住一件事——OCR 的错几乎都是"形近"，不是"义近"。

## 为什么 OCR 错误都是形近的

图片转文字（OCR）是逐字按**字形**猜测的。它看到的是像素，不是含义。所以它会把
"晚"看成"睌"（多了一笔少了一笔，整体轮廓像），但**绝不会**把"机器"看成"机械"——
因为后者是两个字形完全不同的词，OCR 没有任何机制会产生这种混淆。

> 判定主轴：**正确字与错字"长得像不像"。** 像 → 可能是 OCR 错，进一步确认。
> 不像 → 一定不是 OCR 错，不要碰（哪怕你觉得用词不够好）。

## 该改的几类（确信时）

### 1. 形近单字混淆（最常见）

| 错 | 对 | 说明 |
|---|---|---|
| 自已 | 自己 | 己/已/巳 一族，最高频 |
| 末来 | 未来 | 末/未 一横长短 |
| 戊戌 类混用 | 视上下文 | 戊/戍/戌 三兄弟 |
| 睛朗 | 晴朗 | 目/日 偏旁错认 |
| 辩子 | 辫子 | 辩/辨/辫 |
| 候车室 写成 侯车室 | 候 | 候/侯 多一竖 |

### 2. 繁简混入

简体正文里个别字被认成繁体：终→終、关→関、发→發、real例如"門/门""車/车"。
**前提**：整本书是简体语境。若书本身是繁体或繁简混排（如古籍影印），则**不要动**。

### 3. 偏旁/部件级错误

整字因为某个部件被认错而错：如"江"认成"红"（氵/纟）。需要你确认在上下文里
正确字才说得通，才改。

## 不该改的几类（重要）

### 1. 义近形不近的用词差异 —— 绝不碰

- "机械学习" vs "机器学习"：字形完全不同，**不是 OCR 错**，是语义/用词，不改。
- "突然" vs "忽然"、"漂亮" vs "美丽"：同义替换，**不是你的职责**。
- "的/地/得"：虽是常见语病，但**不是 OCR 错误**（字形不混），默认不改。除非用户
  明确要求做语法校正——本技能默认只清扫 OCR 噪声。

### 2. 生僻字、古语、专名、方言 —— 默认保留

看起来"不常见"不等于"错"。人名"喆""昇"、地名、古文用字、作者刻意的方言写法，
一律视为正确，除非你能确证它是某个常见字的形近误认且上下文强烈支持。

### 3. 标点与排版 —— 默认不动

OCR 偶尔会丢标点或认错标点，但判断它"应该是什么"往往需要猜作者意图。除非是
明显乱码（正文里冒出 `□`、控制字符、明显错位的符号），否则不改。

### 4. 模棱两可 —— 一律不改

只要存在"也许作者本来就想这么写"的合理可能，就**不改**。例子：
- "他楞住了" / "他愣住了"：楞、愣常通用，拿不准就保留。
- "唯一" / "惟一"：异形词，二者都对，不改。

## 判别流程（每遇到一个可疑处，过一遍）

1. 正确字和当前字**长得像吗**？不像 → 停，不改。
2. 像。那么当前字在这个上下文里**讲得通吗**？讲得通（只是用词不同）→ 不改。
3. 讲不通，且换成那个形近字就通顺了 → 很可能是 OCR 错。
4. 你能用一句话写出确切理由吗（"X 是 Y 的形近误认"）？能 → 改；不能 → 不改。
5. 这个改动幅度小吗（通常就一两个字）？大改动（换整个词、改语序）→ 不改，那不是 OCR 错。

## 写 fixes 时的注意

- `original` 片段要**足够长以唯一**。例如错字"已"在一章里可能出现几十次，
  只写"已"会无法定位；要写出整句，如"他想起自已年轻时候的事情"。
- 一个 `original` 里若有多个错字，可以在 `corrected` 里一并改正，但 `wrong`/`right`
  只填其中一处时会触发一致性检查失败；建议**一处错字对应一条 fix**，最清晰也最安全。
- 写不出 `reason` 的改动，说明你自己也没把握——删掉它。

## 护栏行为说明（apply_fixes.py 会做的保护）

即便你不小心提了不稳妥的改动，回写脚本也会兜底：
- 片段在该章**出现多次** → 跳过（定位不唯一）。
- 片段在该章**找不到** → 跳过（疑似片段抄错）。
- 改动**幅度过大**（差异超过约 6 字）→ 跳过（疑似过度修改）。
- `original`/`corrected` 与 `wrong`/`right` **对不上** → 跳过（前后不一致）。

被跳过的改动会全部进入 `changes_report.md` 的"未自动应用"区，请提醒用户人工过目。
这套护栏与"少改原则"是一致的：当机器也"拿不准"时，它选择不动手。

# 第二类：旧用字现代化（默认只列出，不应用）

上面讲的都是「**错字**」。这一节讲一类**不是错字、但现代读者可能觉得别扭甚至误以为是错字**的
东西——旧译本/老版本的**用字习惯**。它们合乎当年的正字法，本身没错，但 2026 年的读者读起来硌眼。

**与 OCR 错字的根本区别：**

| | OCR 错字（第一类） | 旧用字（第二类） |
|---|---|---|
| 本质 | 扫描/识别的**错误** | 当年正确的**用字习惯** |
| 信号 | 罕见 + 零散 + 有形近替代 | 常见 + **全书系统性一致** + 符合时代 |
| 处理 | **默认改掉** | **默认只列出**，用户点头才改 |
| 例子 | 莨蓿→莨菪、傍睌→傍晚 | 象→像、馀→余 |

> 注意那个「系统性一致」的判据：如果某个写法**全书统一**地这么用，几乎一定是用字习惯而非随机
> 扫描错误（扫描错误是零散、不规律的）。这正是判断它属于第二类、而非第一类的关键。

## 最典型：象 / 像

老版本常把本该写「像」的地方写成「象」：「好**象**」「**象**空气一样」「**象**小孩子似的」。
现代规范区分「象」（大象、象征）与「像」（好像、图像），所以现代读者看到「好象」会觉得是错字。

**⚠️ 致命陷阱：「象」是多义字，绝不能无脑全文替换。** 必须**逐处判断每个「象」是不是当「像」用**：

- 当「像」用 → 列入现代化候选：好**象**、**象**……一样、**象**……似的、好**象**是
- 本义「象」 → **绝对不列、不动**：大**象**、**象**征、现**象**、气**象**、**象**棋、想**象**、印**象**、对**象**

一个反面教材：`sed 's/象/像/g'` 会把「想象」变成「想像」、「大象」变成「大像」——灾难。
所以这不是查找替换，是**对每一个「象」单独做语义判断**，只挑当「像」用的。

## 其他可考虑的旧用字

- **馀 → 余**（「其馀」→「其余」）、**麽 → 么**（「这麽」→「这么」）等异体字，同样要看是否系统性、是否有歧义。
- 把握不准的、可能两可的（如「唯一/惟一」「身分/身份」），**一律归入存疑，不要硬塞进现代化清单**。
  现代化清单只放「现代读者明显会觉得别扭、且你确信是旧写法」的。

## 怎么写 modernization 的 fix

- `category` 填 `"modernization"`。
- `original` 片段**只包含目标那一个**「象」（或其他目标字）。例如选「所以就象空气一样的自由」，
  **不要**选含「想象」的更长片段——否则一致性检查（`original.replace(wrong,right)==corrected`）会因为
  把「想象」也换掉而失败，导致这条被跳过。
- 一个目标字一条 fix，最清晰。
- `reason` 写明「此处『象』当『像』用」，并可附一句「本义象如大象/象征不在此列」提醒。

## 交互流程（再次强调）

1. 第一遍 `apply_fixes.py` **不加** `--apply-modernization`：OCR 错字改掉，旧用字**只列进报告**。
2. **用中文**问用户：「全书有 N 处『象』是当『像』用的，要不要统一改成现代写法？（大象、象征不会动。）」
3. 用户**同意** → 加 `--apply-modernization` 重跑；**不同意** → 保持，建议仅留在报告供参考。

决定权始终在用户手里——我们只负责把候选**准确地挑出来、摆清楚**。
